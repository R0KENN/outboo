# Outboo — Telegram-бот для модерации и автопостинга

Модульный бот-движок на aiogram 3 для автоматической модерации чатов и отложенной публикации в каналы. Один экземпляр обслуживает множество чатов — у каждого свои настройки, словари и модераторы.

## О проекте

**Outboo** автоматизирует две рутинные задачи администратора Telegram-сообществ — модерацию чатов и ведение каналов — и закрывает их в одном движке.

Проект построен по модульному принципу: каждый функциональный блок (модерация, постинг, статистика) включается и настраивается под конкретный чат без правки кода. Все настройки хранятся в базе данных отдельно для каждого чата, поэтому один запущенный бот обслуживает любое количество групп и каналов одновременно.

> Главная идея: добавили бота в чат, открыли `/settings`, включили нужные модули галочками. Никакого редактирования конфигов под каждую группу.

## Возможности

**Модерация**
- Антиспам: ссылки, упоминания каналов, пересланные сообщения
- Белый список разрешённых доменов
- Антимат сквозь маскировку (пробелы, замены букв, повторы)
- Антифлуд с автоматическим мутом
- Система предупреждений с автодействием на пороге
- Ручные команды наказаний

**Новые участники**
- Капча двух типов: кнопка или математический пример
- Кик по таймауту, если капча не пройдена
- Приветствие с подстановкой имени `{name}` и автоудалением
- Карантин новых аккаунтов
- Очистка служебных сообщений о входе и выходе

**Автопостинг**
- Пошаговый сбор поста в личке с ботом
- Текст, фото, видео, документы, аудио и альбомы
- HTML-форматирование и inline-кнопки со ссылками
- Таймер автоудаления опубликованного поста
- Публикация в несколько каналов из одного бота
- Очередь: просмотр, отмена, перенос времени

**Администрирование**
- Панель настроек на inline-кнопках, отдельно для каждого чата
- Роли младших модераторов с ограниченным набором прав
- Журнал всех модераторских действий

**Статистика**
- Новые участники, активность, удаления по фильтрам, баны, муты, варны
- Период: день, неделя, месяц

**Надёжность**
- Восстановление отложенных задач после перезапуска
- Запуск как systemd-сервис с автоперезапуском
- Логирование с ротацией
- Защита администраторов от случайного бана

## Быстрый старт

### Шаг 1. Создать бота в Telegram

Откройте @BotFather, создайте бота и сохраните токен.

Обязательно выполните `/setprivacy` и выберите **Disable**. Без этого бот не увидит сообщения в группах, и модерация не заработает.

Свой числовой ID узнайте у @userinfobot — он понадобится для переменной `BOT_ADMINS`.

### Шаг 2. Установить проект

```bash
git clone https://github.com/R0KENN/outboo.git
cd outboo

python3.11 -m venv venv
source venv/bin/activate          # Windows: venv\Scripts\activate

pip install -r requirements.txt
```

### Шаг 3. Настроить окружение

```bash
cp .env.example .env
```

Откройте `.env` и заполните два обязательных поля:

```env
BOT_TOKEN=123456789:AAExxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxx
BOT_ADMINS=123456789
```

Остальные параметры можно не трогать — по умолчанию используется локальный файл SQLite `bot.db`.

### Шаг 4. Инициализировать базу и запустить

```bash
alembic revision --autogenerate -m "initial schema"
alembic upgrade head

python bot.py
```

Не хотите возиться с Alembic? Пропустите обе команды `alembic` — при первом запуске бот создаст таблицы автоматически.

### Шаг 5. Подключить к чату

1. Добавьте бота в группу администратором (права: удаление сообщений, блокировка и ограничение участников).
2. Напишите в чате `/settings` и включите нужные модули.
3. Для постинга добавьте бота администратором канала с правом публикации.

Готово.

## Переменные окружения

Все настройки задаются в файле `.env`.

| Переменная          | Назначение                               | По умолчанию |
| ------------------- | ---------------------------------------- | ------------ |
| `BOT_TOKEN`         | Токен бота от @BotFather                  | обязательно  |
| `BOT_ADMINS`        | ID владельцев бота через запятую          | обязательно  |
| `DB_DRIVER`         | Драйвер БД: `sqlite` или `postgres`       | `sqlite`     |
| `SQLITE_PATH`       | Путь к файлу БД для SQLite                 | `bot.db`     |
| `POSTGRES_HOST`     | Хост PostgreSQL                           | `localhost`  |
| `POSTGRES_PORT`     | Порт PostgreSQL                           | `5432`       |
| `POSTGRES_USER`     | Пользователь PostgreSQL                    | `botuser`    |
| `POSTGRES_PASSWORD` | Пароль PostgreSQL                         | —            |
| `POSTGRES_DB`       | Имя базы данных                            | `botdb`      |
| `LOG_LEVEL`         | Уровень логирования                       | `INFO`       |
| `LOG_FILE`          | Путь к файлу логов                         | `logs/bot.log` |
| `THROTTLE_RATE`     | Базовый антифлуд (сек между сообщениями)   | `0.7`        |

## Подключение к Telegram

1. Создайте бота в @BotFather командой `/newbot`, сохраните токен.
2. Отключите приватность: `/setprivacy`, выберите бота, нажмите **Disable**.
3. Добавьте бота в группу администратором с правами: удаление сообщений, блокировка участников, ограничение участников.
4. Для постинга добавьте бота администратором канала с правом публикации сообщений.
5. В группе выполните `/settings` и включите нужные модули.

## Справочник команд

Полный список всегда доступен по команде `/help`.

### Управление чатом (администратор группы)

| Команда               | Описание                                            |
| --------------------- | --------------------------------------------------- |
| `/settings`           | Открыть панель настроек (фильтры и параметры)        |
| `/setwelcome <текст>` | Задать приветствие (`{name}` подставит имя новичка)  |
| `/setrules <текст>`   | Задать правила (добавляются к приветствию)           |

### Наказания (админ/модератор, ответом на сообщение нарушителя)

| Команда            | Описание                          |
| ------------------ | --------------------------------- |
| `/ban`, `/unban`   | Бан и снятие бана                 |
| `/kick`            | Выгнать из чата (сможет вернуться) |
| `/mute 30m`, `/unmute` | Мут на срок и размут           |
| `/warn`, `/unwarn` | Выдать / снять предупреждение      |
| `/warns`           | Показать число предупреждений      |
| `/resetwarns`      | Обнулить предупреждения            |

Срок мута указывается суффиксами: `s` (сек), `m` (мин), `h` (часы), `d` (дни). Например: `/mute 2h`, `/mute 1d`.

### Словари и команда (администратор группы)

| Команда                                | Описание                          |
| -------------------------------------- | --------------------------------- |
| `/addword`, `/delword`, `/words`       | Управление стоп-словами            |
| `/adddomain`, `/deldomain`, `/domains` | Белый список разрешённых доменов    |
| `/addmod mute warn`, `/delmod`, `/mods`| Назначение модераторов и их прав    |
| `/log`                                 | Журнал действий модерации          |

### Статистика (админ/модератор)

| Команда         | Описание           |
| --------------- | ------------------ |
| `/stats day`    | Отчёт за день      |
| `/stats week`   | Отчёт за неделю    |
| `/stats month`  | Отчёт за месяц     |

### Автопостинг (в личке с ботом)

| Команда                            | Описание                                |
| ---------------------------------- | --------------------------------------- |
| `/newpost`                         | Собрать отложенный пост (пошаговый диалог) |
| `/queue`                           | Очередь запланированных постов           |
| `/cancelpost <id>`                 | Отменить пост                            |
| `/repost <id> ДД.ММ.ГГГГ ЧЧ:ММ`    | Перенести время публикации               |

## Описание модулей

### Модерация

Антиспам удаляет ссылки, упоминания каналов и пересланные сообщения. Ссылка на домен из белого списка пропускается. Антимат сверяет текст со словарём стоп-слов и снимает типичные приёмы обхода — пробелы между буквами (`с п а м`), замену букв символами (`сп@м`), растянутые повторы (`спааам`). Антифлуд считает сообщения в скользящем окне времени и автоматически мутит за превышение. Система предупреждений копит варны; при достижении порога срабатывает заданное в настройках действие — мут или бан.

### Новые участники

На входе бот ограничивает право писать и показывает капчу — кнопку «Я не бот» или пример на сложение. Прошёл — получает доступ и приветствие; не успел за отведённое время — отправляется в кик. Карантин держит свежие аккаунты на коротком поводке первые N часов: запрещены ссылки, медиа и форварды. Служебные сообщения о входе и выходе автоматически подчищаются.

### Автопостинг

Пошаговый диалог собирает пост: канал, содержимое, кнопки, время, автоудаление. Принимаются текст, фото, видео, документы, аудио и альбомы; форматирование сохраняется, можно навесить inline-кнопки со ссылками. Планировщик переживает перезапуск бота — задачи восстанавливаются из базы, ничего не теряется. Один бот публикует в любое число каналов.

### Администрирование

Настройки живут в inline-панели и хранятся отдельно для каждого чата. Главный администратор раздаёт младшим модераторам ограниченные права (например, только мут и варн). Каждое действие модерации фиксируется в журнале: кто, что, над кем и когда.

### Статистика

Новые участники, активность, удалённые сообщения по типам фильтров, баны, муты и варны — суммарно за выбранный период (день, неделя, месяц).

## Развёртывание на сервере (systemd)

На Linux-VPS бот работает как systemd-сервис: автоматически перезапускается после сбоя и поднимается при перезагрузке сервера.

1. Установите проект (шаги 2–4 из «Быстрого старта»), например в `/opt/outboo` под пользователем `botuser`.

2. Создайте unit-файл `/etc/systemd/system/outboo.service`:

```ini
[Unit]
Description=Outboo Telegram bot
After=network-online.target
Wants=network-online.target

[Service]
Type=simple
User=botuser
WorkingDirectory=/opt/outboo
ExecStart=/opt/outboo/venv/bin/python /opt/outboo/bot.py
Restart=always
RestartSec=5
Environment=PYTHONUNBUFFERED=1

[Install]
WantedBy=multi-user.target
```

3. Запустите и включите автозагрузку:

```bash
sudo systemctl daemon-reload
sudo systemctl enable --now outboo

sudo systemctl status outboo
journalctl -u outboo -f
```

## Переход на PostgreSQL

Благодаря ORM смена базы не затрагивает бизнес-логику. В `.env`:

```env
DB_DRIVER=postgres
POSTGRES_HOST=localhost
POSTGRES_PORT=5432
POSTGRES_USER=botuser
POSTGRES_PASSWORD=ваш_пароль
POSTGRES_DB=botdb
```

Затем накатите схему:

```bash
alembic upgrade head
```

Код менять не нужно.

## Архитектура

Проект следует строгому трёхслойному разделению ответственности:

```text
HANDLERS  --(события)-->  SERVICES  --(логика)-->  CRUD
ловят и                   вся бизнес-              единственный
вызывают                  логика                   доступ к БД
```

- **handlers** — принимают события Telegram и вызывают сервисы, не содержат логики;
- **services** — хранят всю бизнес-логику (наказания, фильтры, планировщик);
- **crud** — единственное место, где происходит работа с базой данных.

Такое деление обеспечивает переиспользование модулей между проектами и безболезненный переход SQLite → PostgreSQL без изменения логики.
